## 📦 Project Portfolio

### ⬅️ Phase 2 — Requirements Engineering Automation *(Bosch · 2023–Present)*

#### 📥 Step 1 · PDF → RTF → DOORS Import Converter
> *"The tool nobody talks about — but every requirements engineer desperately needs."*

**Problem**: Manual PDF reformatting for IBM DOORS  
**Solution**: Automated pipeline with human-in-loop  
**Impact**: ~70% effort reduction in document preparation  
**Tech**: Python · PDF Parsing · RTF Generation · DOORS  
**Status**: ![Internal](https://img.shields.io/badge/INTERNAL-confidential) – Deployed internally

#### 🔍 Step 2 · AI Requirement Similarity Assistant
> *"Stop re-inventing requirements that already exist."*

**Problem**: 15-day manual redundancy checks across 10+ legacy projects  
**Solution**: RAG pipeline with exact-match filter (40% token reduction)  
**Impact**: **15 days → 5 days (65% lead-time cut)** + **$8,880 cost avoidance + 280 hrs saved**  
**Tech**: Python · Azure OpenAI GPT-4 · RAG · FAISS · Streamlit  
**Status**: ![Live](https://img.shields.io/badge/LIVE-green) + [Public POC](https://github.com/Vignesh-Manivasakam/Requirement-Similarity-Assistant-POC)

#### ⚙️ Step 3 · ML-Based Requirement Allocator *(In Progress)*
> *"Who should own this requirement?"*

**Problem**: Inconsistent manual allocation across OEMs  
**Solution**: NLP classifier with confidence scoring  
**Tech**: Python · NLP · Transformers · LLM  
**Status**: ![In Progress](https://img.shields.io/badge/IN%20PROGRESS-orange) – Model training ongoing

#### 🛡️ Step 4 · Agentic Multi-Domain Requirement Reviewer *(POC)*
> *"Safety expert + Test engineer + Architect in one system."*

**Problem**: Missed safety attributes in manual reviews  
**Solution**: Multi-agent system (Safety & Security Agent for ISO 26262/SOTIF, Test Verifiability Agent)  
**Impact**: **>90% recall** vs manual domain-expert review  
**Tech**: Python · Multi-Agent Systems · Prompt Engineering · LangGraph  
**Status**: ![POC Built](https://img.shields.io/badge/POC%20BUILT-blue) – Validated internally
